1. Introduction

This project compares hyperparameter optimization algorithms for machine learning models. The goal is to evaluate the effectiveness and efficiency of Grid Search, Random Search, and Bayesian Optimization methods.

2. Optimization Algorithms

2.1 Grid Search

ðŸ“Š Algorithm: Grid Search

Type: Exhaustive search

How it works: Systematically tries all possible parameter combinations. Evaluates each with cross-validation.

Advantages: Guaranteed to find global optimum within search space

Disadvantages: Exponentially slow as parameters increase (curse of dimensionality)

2.2 Random Search

ðŸŽ² Algorithm: Random Search

Type: Stochastic search

How it works: Samples random points from parameter space. Tries for specified iterations.

Advantages: Much faster than Grid Search, effective in high-dimensional spaces

Disadvantages: No guarantee of finding optimum

Reference: Bergstra & Bengio (2012)

2.3 Bayesian Optimization (Optuna)

ðŸ§  Algorithm: Bayesian Optimization

Type: Model-based search

How it works: Learns from past evaluations using Tree-structured Parzen Estimator (TPE). Intelligently selects next trial point.

Advantages: Sample efficient, good results with few trials

Library used: Optuna (TPE algorithm)

4. Conclusions

ðŸŽ¯ Key Findings:
  1. Bayesian always found the best score: Consistently outperformed Grid and Random Search
  2. Random Search surprisingly effective: Same score as Grid Search in half the time
  3. Computation trade-off: Bayesian gives better results but 3-10x slower

Recommendations

  • Limited resources: Use Random Search - fast and effective
  • Critical applications: Use Bayesian Optimization - best results
  • Small parameter spaces: Grid Search is interpretable
